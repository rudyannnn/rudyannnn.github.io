## 이 문서는?
맥미니에서 이 블로그를 클론해서 처음으로 직접 작성해 본 글이다.
글쓰기 흐름을 익히기 위한 연습 겸 참고 메모.

## 새 글을 쓰는 순서
1. `_wiki/` 폴더에 `문서이름.md` 파일을 만든다.
2. 파일 맨 위에 `---` 로 둘러싼 frontmatter 를 넣는다. (제목, 날짜, tag, public 등)
3. 본문은 마크다운으로 쓴다.
4. 다른 문서로 링크할 때는 `[[문서이름]]` 형식을 쓴다.
5. 저장 후 `git add` → `git commit` → `git push` 하면 끝.
6. 잠시 뒤 `https://rudyannnn.github.io` 에 자동 반영된다.

## frontmatter 항목 뜻
* `title`   : 글 제목 (비워두고 본문 `#` 제목을 써도 된다)
* `date`    : 최초 작성일 (`+0900` 은 한국 시간)
* `updated` : 마지막 수정일
* `tag`     : 태그. 공백으로 여러 개 구분
* `public`  : `true` 면 공개, `false` 면 비공개
* `parent`  : 상위 문서. 보통 `[[index]]`

## 파일명 규칙 (중요! 404 방지)
GitHub Pages 에서 페이지가 404 나는 걸 겪고 얻은 규칙이다.

* **순수 ASCII 권장** : `ci-cd-gem-docker.md` 처럼. 주소가 깔끔하고 안 깨진다.
* **순수 한글도 OK** : `독서록.md` 처럼.
* **대문자 ASCII + 한글 혼합은 금지** : `CI-CD-gem-도커이미지.md` 같은 이름은
  sitemap 에는 등록되지만 실제 페이지가 **404** 가 됐다. (순수 한글 글은 멀쩡했음)
* 한글 제목은 frontmatter 의 `title` 에 적으면 페이지 제목으로 잘 나온다.
* 목차에서 한글로 예쁘게 보이고 싶으면 표시텍스트 문법을 쓴다:
  `[[ci-cd-gem-docker]]{CI, CD, gem, 도커 이미지}`
  → 파일은 ASCII, 화면에는 한글로 보인다. (`_includes/createLink.html` 이 지원)

## 기억할 점
* `[[ ]]` 링크는 vimwiki 방식이라 파일 이름과 정확히 같아야 한다.
* `[[파일명]]{표시텍스트}` 로 링크 글자를 따로 지정할 수 있다.
* `_wiki/index.md` 의 목차에 `* [[맥미니에서-블로그-쓰기]]` 를 추가하면
  첫 페이지에서도 이 글로 갈 수 있다.
